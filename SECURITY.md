# Security Policy

## Supported Versions

Security fixes are applied to the latest state of the `main` branch.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

To report a vulnerability:

1. Open a GitHub issue and clearly mark it as a **private security report request**, or
2. Contact the repository owner directly through GitHub.

When reporting, include:

- A clear description of the issue
- Steps to reproduce
- Potential impact
- Any suggested remediation

We aim to acknowledge reports quickly and provide a remediation plan after triage.
